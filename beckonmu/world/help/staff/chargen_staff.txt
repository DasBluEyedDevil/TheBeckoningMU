|[R+===================================================================+
|W|               STAFF CHARACTER APPROVAL COMMANDS                  |
|[R+===================================================================+|n

Commands for staff to review, edit, approve, and reject character
applications submitted through the character generation system.

|WLISTING PENDING CHARACTERS:|n

|y+pending|n
  List all characters awaiting staff approval.

Shows:
  - Character name
  - Player name
  - Clan
  - Time since submission

Characters are sorted oldest-first to help process applications
in order.

|WREVIEWING A CHARACTER:|n

|y+review <character name>|n
  Display a full character sheet for review.

|yExample:|n
  +review John

Shows complete character information:
  - Basic info (name, concept, clan, generation, sire, predator type)
  - Physical, Social, and Mental Attributes
  - Physical, Social, and Mental Skills
  - Disciplines
  - Specialties
  - Approval status and submission time

|WEDITING CHARACTER TRAITS:|n

|y+charedit <character>/<field>=<value>|n
  Edit a character's traits and information.

|yBio Fields:|n
  name, concept, clan, generation, sire, predator_type

|yTrait Fields:|n
  Any attribute, skill, or discipline name

|yExamples:|n
  +charedit John/clan=Brujah
  +charedit Mary/strength=3
  +charedit Bob/academics=2
  +charedit Sarah/celerity=2

All edits are logged with staff member name, timestamp, and the
change made. A reason prompt may appear for audit purposes.

|WAPPROVING A CHARACTER:|n

|y+approve <character>[=<message>]|n
  Approve a character for play.

|yExamples:|n
  +approve John
  +approve Mary=Great character! Welcome to the game.

When approved:
  - Character is marked as approved with your name and timestamp
  - Player receives a notification that they can begin playing
  - Optional message is included in the notification
  - Related Jobs system ticket is closed automatically

|WREJECTING A CHARACTER:|n

|y+reject <character>=<reason>|n
  Reject a character application with feedback.

|yExamples:|n
  +reject John=Skills exceed allowed limits. Please review chargen rules.
  +reject Mary=Clan choice doesn't match concept. Please revise.

A reason is |WREQUIRED|n when rejecting a character.

When rejected:
  - Player receives a notification with your feedback
  - Character remains in pending status
  - Player can make changes and resubmit
  - Related Jobs system ticket is updated with feedback

|WIMPORTING CHARACTERS:|n

|y+import <filename>|n
  Import character data from a JSON file.

Used primarily for importing characters created via the web
character generator.

|yRequirements:|n
  - File must be in server/conf/character_imports/
  - File must be valid JSON
  - You must be puppeting the character to import to

|yExamples:|n
  +import mycharacter.json
  +import john_brujah.json

|WWORKFLOW RECOMMENDATIONS:|n

1. Check |y+pending|n regularly for new applications
2. Use |y+review|n to examine each character thoroughly
3. Verify traits match concept and clan appropriately
4. Check for any obvious power-gaming or rule violations
5. Use |y+charedit|n for minor corrections if needed
6. Approve with a welcoming message, or reject with clear feedback
